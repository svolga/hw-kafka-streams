# hw-kafka-streams
## Задача. 
Создайте систему обработки потоков сообщений с функциями блокировки пользователей и цензуры сообщений. 
Разверните систему с использованием Docker сompose и настройте необходимые топики Kafka.

## Решение
1. В каталоге docker запустить docker-compose.yaml файл c сервисами: zookeeper, kafka broker, kafka ui

```
     docker compose up -d
```


2. В приложении создаются топики с сообщениями, заблокированными пользователями,
словами для фильтрации, отфильтрованными сообщениями:

#### messages, blocked_users, forbidden-words, filtered_messages.


3. Классы приложения: record для хранения сообщений, serde - сериализатор/десериализатор обьекта в массив байт и обратно,
процессор с имплементаций интерфейса ValueTransformer для изменения каждой записи. 


4. Тестовые данные.
- Массив запрещенных слов:

```
     {"Политика", "1C", "Алкоголь"}
```

- Массив блокированных логинов пользователей:

```
      {"login1:login2", "login1:login3", "login2:login4"}
```


- Массив c данными ({автор} ->{{текст, получатель}}): 

```
  {
       login2 -> {"Spring", "login1"},
       login3 -> {"1С", "login1"},
       login4 -> {"Java", "login1"}
       login5 -> {"Политика React", "login1"
  }
```

## Результаты
   Создается топик filtered_messages с набором ключ (автор)->значение ({текст, получатель})

```
     login4 -> { "text": "Java", "receiver": "login1" }
     login5 -> { "text": "******** React", "receiver": "login1" }
```
